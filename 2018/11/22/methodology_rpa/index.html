方法论：企业应该如何正确把握RPA

发表于 2018-11-22 | 分类于 Robotic Process Automation
字数统计: 1.5k | 阅读时长 ≈ 5

RPA（Robotic Process Automation）即机器人流程自动化，这是一种全新的虚拟机器人概念，意在模拟人类与计算机的交互，代替人工处理复杂、繁琐以及大量的各项事务，从而大量减少企业的人力成本 ，提升整体工作效率，甚至能够辅助发现工作流程中不必要的环节，实现流程优化。RPA，无疑是企业向数字化转型的重要技术之一。

既然如此 ，那应该如何去正确的处理，将RPA成功落实到企业中去？企业又应该从哪些方面着手考虑，让RPA为企业带去极为可观助力？

RPA三步走

随着RPA在市场上的崛起，越来越多的企业成功的将RPA这门技术引入到生产环节上，坐享其带来的好处，大大提升了企业的核心竞争力，这其中不泛有AIG、大都会人寿 Met Life， Allstate等知名企业，在国内，RPA更是在财务、金融、制造业、通讯行业等，大行其道。

选择正确的自动化流程是获得最高投资回报率（Return On Investment,ROI）的关键因素，用户必须要为“机器人员工”安排恰当的工作范围。

小金人在经过这三年来不断的业务磨练，以及在行业内积累的丰富经验，提出以下六个RPA关键步骤，并附以矩阵评估，从而让RPA找到最合适的位置：

  • 充分论证

    根据具体需求以及现有资源，进行明确的业务范围界定。对现有流程，以及按照RPA的工作特性，对RPA的建设目标以及效果进行充分评估，以组织和流程的改造层面，制定ROI，为RPA机器人的映入进行可信性论证。

  • RPA的落地场景

    充分理解RPA机器人的特性，主要围绕以下特点，寻找机器人的落地场景，RPA机器人处理场景具备以下特点：

    1. 处理高度可重复任务：通过软件编程语言实现的机器人可以处理重复的人工任务；
    2. 基于明确的规则操作：流程必须有明确的、可被数字化的触发指令和输入，流程不得出现无法提前定 义的例外情况；
    3. 以外挂形式进行部署：基于规则在用户界面进行自动化操作，非侵入式模式 不影响原有IT基础架构；

    4. 模拟手工操作及交互：机器人可以执行用户日常基本操作，例如：鼠标点击、键盘输入、复制/粘 贴等一系列日常电脑操作；

      基于以上特点的理解，识别出目前业务流程中可以用机器人代替的工作场景，工作场景的落地主要完成以下 几个方面的分析：

    5. 现状分析：分析流程中可以使用RPA的工作场景，RPA的主要候选任务就是那些需要大量重复的任务，相关场景具备操作流程基本规则化、重复化的工作特征；

    6. 流程分析：对工作场景进行流程化分析，需要标准化的操作步骤，以及操作步骤执行结果的检查点，制定流程操作及检查结果的数字化标准，数字化标准包括流程的人工操作步骤、操作的输入参数、输出结果，流程执行预期结果及检查规则

  • POC测试

    制定方案，首要确定最有可能看到具有积极业务影响的流程，通过ROI分析，挑选最优选择，确保提升实现业务流程现代化的可能性，从而部署后获得最大价值。

企业必须要考虑RPA部署后的可维护性，这是重要测试指标之一，RPA部署必须具备较强的可维护性，RPA操作脚本必须具备参数化调整，同时业必须提供模块化组件，确保在系统调整时候能够快速响应，易于维护。

同时考虑供应商是否有合理丰富的安全机制，可以保证RPA部署后的系统安全性。

在安全方面，请确保有如下的安全机制：

  1. RPA机器人必须确保所有操作可监控、结果可审计，提供对执行预期结果的定义与检查；
  2. 提供异常中断报警机制；
  3. 一旦发生流程执行异常，操作人员可即时接管故障节点进行故障处理，不需再进行故障物理定位；
  4. 执行结果全程可视化，支持系统日志、流程执行、流程关键节点和操作截屏、操作全程录像，事后可以查询和审计；

在整个实施过程中，关键需要让管理和运营团队参与进来，以确保有效的支持，承诺以及协作，并促进采用。

RPA流程选定评分矩阵

确定RPA的最佳候选者绝对至关重要，但RPA仍应被视为端到端流程改进的一部分。 RPA的引入必须为进一步的自动化和进一步提高效率提供机会。

4点关于RPA项目的考虑

  1. 转型战略调整

    确保RPA被纳入您的全局转型战略，并考虑到其他转型手段；

  2. RPA范围界定和评估

    通过适当的方法筛选您的方案，以评估符合条件的流程/任务，并定义相关的业务，包括定量和定性预期收益

  3. RPA的POC实施

    选择一个采购方案，协助实施试点，并在更大范围内推出解决方案之前分析输出

  4. RPA治理和运营模式定义

    定义适当的治理，并支持变更管理以及积极沟通，确保员工的支持
